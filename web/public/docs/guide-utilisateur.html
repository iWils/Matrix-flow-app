Guide Utilisateur Matrix Flow

Bienvenue dans Matrix Flow ! Ce guide vous accompagnera dans la découverte et l'utilisation de toutes les fonctionnalités de l'application de gestion des matrices de flux réseau.

Connexion à l'application

Première connexion

1
Accédez à l'application
Ouvrez votre navigateur et rendez-vous à l'adresse fournie par votre administrateur.
2
Saisissez vos identifiants
Entrez votre nom d'utilisateur et mot de passe fournis par votre administrateur.
3
Configurez votre profil
Lors de votre première connexion, vous serez invité à changer votre mot de passe et à configurer vos préférences.
Sécurité : Changez votre mot de passe par défaut dès votre première connexion pour garantir la sécurité de votre compte.

Interface utilisateur

L'interface de Matrix Flow est composée de plusieurs éléments principaux :

  • Barre de navigation : Accès rapide aux différentes sections
  • Menu utilisateur : Gestion du profil, paramètres et déconnexion
  • Sélecteur de langue : Changement de langue en temps réel
  • Thème : Basculement entre mode clair et sombre

Tableau de bord

Le tableau de bord est votre point d'entrée principal dans l'application. Il affiche :

📊 Statistiques

Nombre de matrices, flux actifs, demandes en attente et activité récente.

🔔 Notifications

Alertes sur les demandes d'approbation, les modifications importantes et les échéances.

⚡ Accès rapide

Liens directs vers vos matrices fréquemment utilisées et les actions courantes.

📈 Activité récente

Historique de vos dernières modifications et actions dans l'application.

Gestion des matrices

Création d'une nouvelle matrice

1
Accédez à la section Matrices
Cliquez sur "Matrices" dans la navigation principale.
2
Créez une nouvelle matrice
Cliquez sur le bouton "Nouvelle matrice" et remplissez les informations requises :
  • Nom : Nom descriptif de votre matrice
  • Description : Description détaillée de l'usage et du contexte
  • Nombre d'approbations requis : Nombre de validations nécessaires pour les modifications
3
Configurez les permissions
Définissez qui peut accéder et modifier votre matrice en ajoutant des utilisateurs avec des rôles spécifiques.

Visualisation d'une matrice

Chaque matrice dispose d'une vue détaillée permettant de :

  • Consulter tous les flux configurés
  • Filtrer et rechercher dans les entrées
  • Exporter les données au format CSV ou JSON
  • Visualiser l'historique des modifications
  • Accéder aux versions précédentes

Modification d'une matrice

Permissions requises : Vous devez avoir le rôle "propriétaire" ou "éditeur" sur la matrice pour pouvoir la modifier.

Les modifications d'une matrice incluent :

  • Changement du nom ou de la description
  • Ajustement du nombre d'approbations requises
  • Configuration des webhooks de notification
  • Gestion des permissions utilisateur

Système de permissions

Rôle Permissions
Propriétaire Contrôle total : modification, suppression, gestion des permissions
Éditeur Modification des flux, création de demandes de changement
Lecteur Consultation uniquement des flux et de l'historique

Gestion des flux

Création d'un flux

Un flux représente une règle de trafic réseau dans votre matrice. Pour créer un nouveau flux :

1
Ouvrez la matrice cible
Accédez à la matrice dans laquelle vous souhaitez ajouter le flux.
2
Cliquez sur "Nouveau flux"
Le formulaire de création s'ouvre avec tous les champs nécessaires.
3
Remplissez les informations du flux
Complétez les champs suivants selon vos besoins :
Champ Description Exemple
Type de demande Nature de la demande de flux Nouveau flux, Modification, Suppression
Nom de la règle Identifiant unique de la règle ALLOW_HTTP_WEB
Équipement Équipement réseau concerné Firewall-01, Router-DMZ
Zone source Zone réseau d'origine du trafic DMZ, LAN, WAN
Nom source Nom descriptif de la source Serveurs Web
CIDR source Adresse réseau source 192.168.1.0/24
Service source Port/service source ANY, TCP/80
Zone destination Zone réseau de destination Internet, DMZ
Nom destination Nom descriptif de la destination Serveurs de base de données
CIDR destination Adresse réseau destination 10.0.0.0/16
Groupe de protocole Type de protocole HTTP, HTTPS, FTP
Service destination Port/service destination TCP/443, UDP/53
Action Action à appliquer ALLOW, DENY, LOG
Date d'implémentation Date prévue de mise en œuvre 2024-03-15
Demandeur Personne ayant fait la demande john.doe@company.com
Commentaire Justification ou remarques Ouverture pour nouvelle application
Validation : Tous les flux sont automatiquement validés avant création. Assurez-vous que les formats d'adresses IP et de ports sont corrects.

Modification d'un flux existant

Pour modifier un flux existant :

  1. Localisez le flux dans la liste ou utilisez la fonction de recherche
  2. Cliquez sur l'icône d'édition (crayon) à droite de la ligne
  3. Modifiez les champs nécessaires dans le formulaire
  4. Ajoutez un commentaire expliquant la raison de la modification
  5. Validez en cliquant sur "Enregistrer les modifications"
Workflow d'approbation : Selon la configuration de la matrice, vos modifications peuvent nécessiter une approbation avant d'être appliquées.

Suppression d'un flux

La suppression d'un flux suit un processus contrôlé :

Attention : La suppression d'un flux est définitive et peut impacter la sécurité de votre réseau. Assurez-vous de la nécessité de cette action.
  1. Sélectionnez le flux à supprimer
  2. Cliquez sur l'icône de suppression (corbeille)
  3. Confirmez l'action dans la boîte de dialogue
  4. Ajoutez une justification obligatoire
  5. La suppression sera soumise au workflow d'approbation si configuré

Workflow d'approbation

Le système de workflow garantit que tous les changements importants sont validés avant leur application.

Types de demandes de changement

🆕 Création de flux

Demande d'ajout d'un nouveau flux dans la matrice.

✏️ Modification de flux

Demande de modification d'un flux existant.

🗑️ Suppression de flux

Demande de suppression d'un flux de la matrice.

États des demandes

État Description Actions possibles
En attente Demande soumise, en attente de révision Modifier, Annuler
Approuvée Demande validée et appliquée Consulter l'historique
Rejetée Demande refusée avec commentaires Réviser et resoumettre

Gestion de vos demandes

Dans la section "Workflow", vous pouvez :

  • Consulter toutes vos demandes en cours et passées
  • Suivre le statut de chaque demande
  • Lire les commentaires des approbateurs
  • Modifier les demandes en attente
  • Annuler des demandes non encore traitées

Import et Export

Export des données

Matrix Flow permet d'exporter vos matrices dans plusieurs formats :

1
Accédez à la matrice
Ouvrez la matrice que vous souhaitez exporter.
2
Sélectionnez le format
Choisissez entre les formats disponibles :
  • CSV : Compatible avec Excel et autres tableurs
  • JSON : Format structuré pour l'intégration système
  • XML : Format standard pour les systèmes d'entreprise
3
Configurez les options
Définissez les filtres et options d'export (période, statut des flux, colonnes à inclure).

Import de données

L'import permet de charger des flux en masse depuis un fichier :

Format requis : Le fichier d'import doit respecter le format template fourni. Téléchargez le modèle avant de préparer vos données.
  1. Téléchargez le template d'import depuis la matrice cible
  2. Remplissez le fichier avec vos données de flux
  3. Validez le format et la cohérence des données
  4. Uploadez le fichier via l'interface d'import
  5. Vérifiez le rapport de validation
  6. Confirmez l'import si tout est correct

Système de versioning

Chaque matrice dispose d'un système de versioning complet qui conserve l'historique de tous les changements.

Fonctionnalités du versioning

  • Versions automatiques : Création automatique lors de modifications importantes
  • Snapshots complets : Chaque version contient l'état complet de la matrice
  • Métadonnées : Informations sur l'auteur, la date, et les raisons du changement
  • Restauration : Possibilité de revenir à une version antérieure
  • Comparaison : Comparaison entre versions pour identifier les changements

Consultation de l'historique

Pour accéder à l'historique des versions :

  1. Ouvrez la matrice concernée
  2. Cliquez sur l'onglet "Versions" ou "Historique"
  3. Parcourez la liste des versions chronologique
  4. Cliquez sur une version pour voir les détails
  5. Utilisez la fonction "Comparer" pour voir les différences

Gestion du profil utilisateur

Informations personnelles

Accédez à votre profil via le menu utilisateur (icône en haut à droite) :

  • Nom complet : Modifiable selon les permissions
  • Email : Utilisé pour les notifications
  • Nom d'utilisateur : Identifiant de connexion
  • Rôle global : Défini par l'administrateur
  • Langue : Interface et notifications

Changement de mot de passe

1
Accédez aux paramètres
Menu utilisateur > "Changer le mot de passe"
2
Saisissez les informations
  • Mot de passe actuel
  • Nouveau mot de passe (minimum 8 caractères)
  • Confirmation du nouveau mot de passe
3
Validez le changement
Le système vérifiera la complexité et mettra à jour votre mot de passe.
Politique de mot de passe : Les mots de passe doivent contenir au minimum 8 caractères avec au moins une majuscule, une minuscule et un chiffre.

Préférences d'interface

  • Langue : Français, Anglais (interface temps réel)
  • Thème : Mode clair ou sombre
  • Notifications : Configuration des alertes par email
  • Fuseau horaire : Affichage des dates et heures

Historique et audit

Matrix Flow maintient un historique complet de toutes les actions pour garantir la traçabilité et la conformité.

Types d'événements audités

Catégorie Actions
Authentification Connexions, déconnexions, tentatives échouées
Matrices Création, modification, suppression, changements de permissions
Flux Ajout, modification, suppression de règles de flux
Workflow Soumission, approbation, rejet de demandes
Profil Modifications du profil, changements de mot de passe
Import/Export Opérations d'import et d'export de données

Consultation de l'audit

Selon vos permissions, vous pouvez consulter :

  • Votre historique personnel : Toutes vos actions dans l'application
  • Historique des matrices : Actions sur les matrices où vous avez des permissions
  • Historique complet : Toutes les actions (administrateurs uniquement)

Filtrage et recherche

L'interface d'audit permet de filtrer les événements par :

  • Période (date de début et fin)
  • Type d'action (création, modification, suppression)
  • Utilisateur (si vous avez les permissions)
  • Matrice ou entité concernée
  • Adresse IP source
Conformité : L'audit trail répond aux exigences de conformité et peut être exporté pour les audits externes.
